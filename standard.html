Housing Transparency Standard

The Housing Transparency Standard defines the minimum information rental listings should disclose before a renter applies. It is designed to make cost, eligibility, and lease information easier to find, compare, and understand.

Purpose of the standard

Rental listings often present housing information in pieces. A renter may see the advertised rent first, then discover additional fees, deposits, utility costs, or qualification requirements later in the process.

This standard organizes the information renters need into five disclosure layers. These layers can be used to evaluate listing transparency and guide more renter-centered housing platform design.

The five disclosure layers

Layer 1: Property Information

Basic listing details should be visible and easy to confirm.

  • Property address or location
  • Unit type
  • Square footage
  • Availability date
  • Lease term options

Layer 2: Financial Transparency

Renters should be able to understand the full recurring monthly cost.

  • Base rent
  • Required monthly fees
  • Utility estimates
  • Parking costs
  • Pet rent or recurring pet fees

Layer 3: Qualification Requirements

Renters should know whether they are likely to qualify before paying to apply.

  • Income requirement
  • Credit requirement
  • Guarantor policy
  • Background screening criteria
  • Application eligibility rules

Layer 4: Lease and Policy Information

Key lease rules should be presented in plain language before commitment.

  • Lease length
  • Renewal policy
  • Renter's insurance requirement
  • Pet policy
  • Community or property rules

Layer 5: Move-In Cost Disclosure

Upfront costs should be summarized before the application process begins.

  • Application fee
  • Administrative fee
  • Security deposit
  • First month's rent
  • Other required move-in charges

How the standard supports renters

The standard helps renters compare listings using complete information rather than relying only on advertised rent. It also reduces the need to search across multiple pages, call leasing offices, or begin an application just to find basic affordability information.

Design implication

A transparent housing interface should not only display information. It should organize information around the decisions renters need to make: Can I afford this unit? Do I qualify? What will I pay monthly? What will I owe before move-in?
